文件系统损坏原因

2014年06月20日

原因：
关于昨天服务器文件系统损坏的原因，我怀疑可能是这样的，上周给服务器装了个ngx_pagespeed，一个用来提高网站性能的nginx模块，它运行时产生大量的cache文件（按默认设置放在系统盘的/var/目录上）。网站高访问量时会对cache文件进行高速读写，可能这样导致系统盘的文件系统损坏。

现在新服务器上把这些cache放在单独的临时文件系统分区，跟系统盘分离了。
